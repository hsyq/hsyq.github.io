# Redis命令

# Redis 键(key)

Redis 键命令用于管理 redis 的键。

Redis 键命令的基本语法如下：

redis 127.0.0.1:6379> COMMAND KEY_NAME

demo:

redis 127.0.0.1:6379> SET runoobkey redis
OK
redis 127.0.0.1:6379> DEL runoobkey
(integer) 1

在以上实例中 DEL 是一个命令， runoobkey 是一个键。 如果键被删除成功，命令执行后输出 (integer) 1，否则将输出 (integer) 0

# Redis keys 命令

下表给出了与 Redis 键相关的基本命令：

序号 命令及描述
1 DEL key (opens new window) 该命令用于在 key 存在时删除 key。
2 DUMP key (opens new window) 序列化给定 key ，并返回被序列化的值。
3 EXISTS key (opens new window) 检查给定 key 是否存在。
4 EXPIRE key (opens new window) seconds 为给定 key 设置过期时间，以秒计。
5 EXPIREAT key timestamp (opens new window) EXPIREAT 的作用和 EXPIRE 类似，都用于为 key 设置过期时间。 不同在于 EXPIREAT 命令接受的时间参数是 UNIX 时间戳(unix timestamp)。
6 PEXPIRE key milliseconds (opens new window) 设置 key 的过期时间以毫秒计。
7 PEXPIREAT key milliseconds-timestamp (opens new window) 设置 key 过期时间的时间戳(unix timestamp) 以毫秒计
8 KEYS pattern (opens new window) 查找所有符合给定模式( pattern)的 key 。
9 MOVE key db (opens new window) 将当前数据库的 key 移动到给定的数据库 db 当中。
10 PERSIST key (opens new window) 移除 key 的过期时间，key 将持久保持。
11 PTTL key (opens new window) 以毫秒为单位返回 key 的剩余的过期时间。
12 TTL key (opens new window) 以秒为单位，返回给定 key 的剩余生存时间(TTL, time to live)。
13 RANDOMKEY (opens new window) 从当前数据库中随机返回一个 key 。
14 RENAME key newkey (opens new window) 修改 key 的名称
15 RENAMENX key newkey (opens new window) 仅当 newkey 不存在时，将 key 改名为 newkey 。
16 [SCAN cursor MATCH pattern] [COUNT count] (opens new window) 迭代数据库中的数据库键。
17 TYPE key (opens new window) 返回 key 所储存的值的类型。

更多命令请参考：https://redis.io/commands

# Redis 字符串(String)

Redis 字符串数据类型的相关命令用于管理 redis 字符串值，基本语法如下：

redis 127.0.0.1:6379> COMMAND KEY_NAME

redis 127.0.0.1:6379> SET runoobkey redis
OK
redis 127.0.0.1:6379> GET runoobkey
"redis"

在以上实例中我们使用了 SET 和 GET 命令，键为 runoobkey。

# Redis 字符串命令

下表列出了常用的 redis 字符串命令：

序号 命令及描述
1 SET key value (opens new window) 设置指定 key 的值。
2 GET key (opens new window) 获取指定 key 的值。
3 GETRANGE key start end (opens new window) 返回 key 中字符串值的子字符
4 GETSET key value (opens new window) 将给定 key 的值设为 value ，并返回 key 的旧值(old value)。
5 GETBIT key offset (opens new window) 对 key 所储存的字符串值，获取指定偏移量上的位(bit)。
6 [MGET key1 key2..] (opens new window) 获取所有(一个或多个)给定 key 的值。
7 SETBIT key offset value (opens new window) 对 key 所储存的字符串值，设置或清除指定偏移量上的位(bit)。
8 SETEX key seconds value (opens new window) 将值 value 关联到 key ，并将 key 的过期时间设为 seconds (以秒为单位)。
9 SETNX key value (opens new window) 只有在 key 不存在时设置 key 的值。
10 SETRANGE key offset value (opens new window) 用 value 参数覆写给定 key 所储存的字符串值，从偏移量 offset 开始。
11 STRLEN key (opens new window) 返回 key 所储存的字符串值的长度。
12 [MSET key value key value ...] (opens new window) 同时设置一个或多个 key-value 对。
13 [MSETNX key value key value ...] (opens new window) 同时设置一个或多个 key-value 对，当且仅当所有给定 key 都不存在。
14 PSETEX key milliseconds value (opens new window) 这个命令和 SETEX 命令相似，但它以毫秒为单位设置 key 的生存时间，而不是像 SETEX 命令那样，以秒为单位。
15 INCR key (opens new window) 将 key 中储存的数字值增一。
16 INCRBY key increment (opens new window) 将 key 所储存的值加上给定的增量值（increment） 。
17 INCRBYFLOAT key increment (opens new window) 将 key 所储存的值加上给定的浮点增量值（increment） 。
18 DECR key (opens new window) 将 key 中储存的数字值减一。
19 DECRBY key decrement (opens new window) key 所储存的值减去给定的减量值（decrement） 。
20 APPEND key value (opens new window) 如果 key 已经存在并且是一个字符串， APPEND 命令将指定的 value 追加到该 key 原来值（value）的末尾。

更多命令请参考：https://redis.io/commands

# Redis 哈希(Hash)

Redis hash 是一个 string 类型的 field（字段） 和 value（值） 的映射表，hash 特别适合用于存储对象。

Redis 中每个 hash 可以存储 232 - 1 键值对（40多亿）。

demo:

127.0.0.1:6379>  HMSET runoobkey name "redis tutorial" description "redis basic commands for caching" likes 20 visitors 23000
OK
127.0.0.1:6379>  HGETALL runoobkey
1) "name"
2) "redis tutorial"
3) "description"
4) "redis basic commands for caching"
5) "likes"
6) "20"
7) "visitors"
8) "23000"

在以上实例中，我们设置了 redis 的一些描述信息(name, description, likes, visitors) 到哈希表的 runoobkey 中。

# Redis hash 命令

下表列出了 redis hash 基本的相关命令：

序号 命令及描述
1 [HDEL key field1 field2] (opens new window) 删除一个或多个哈希表字段
2 HEXISTS key field (opens new window) 查看哈希表 key 中，指定的字段是否存在。
3 HGET key field (opens new window) 获取存储在哈希表中指定字段的值。
4 HGETALL key (opens new window) 获取在哈希表中指定 key 的所有字段和值
5 HINCRBY key field increment (opens new window) 为哈希表 key 中的指定字段的整数值加上增量 increment 。
6 HINCRBYFLOAT key field increment (opens new window) 为哈希表 key 中的指定字段的浮点数值加上增量 increment 。
7 HKEYS key (opens new window) 获取所有哈希表中的字段
8 HLEN key (opens new window) 获取哈希表中字段的数量
9 [HMGET key field1 field2] (opens new window) 获取所有给定字段的值
10 [HMSET key field1 value1 field2 value2 ] (opens new window) 同时将多个 field-value (域-值)对设置到哈希表 key 中。
11 HSET key field value (opens new window) 将哈希表 key 中的字段 field 的值设为 value 。
12 HSETNX key field value (opens new window) 只有在字段 field 不存在时，设置哈希表字段的值。
13 HVALS key (opens new window) 获取哈希表中所有值。
14 [HSCAN key cursor MATCH pattern] [COUNT count] (opens new window) 迭代哈希表中的键值对。

更多命令请参考：https://redis.io/commands

# Redis 列表(List)

Redis列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边）或者尾部（右边）

一个列表最多可以包含 232 - 1 个元素 (4294967295, 每个列表超过40亿个元素)。

redis 127.0.0.1:6379> LPUSH skills coding
(integer) 1
redis 127.0.0.1:6379> LPUSH skills writing
(integer) 2
redis 127.0.0.1:6379> LPUSH skills runnig
(integer) 3
redis 127.0.0.1:6379> LRANGE skills 0 3
# 获取0，1，2这三个value
1) "mysql"
2) "mongodb"
3) "redis"

在以上实例中我们使用了 LPUSH 将三个值插入了名为 runoobkey 的列表当中。

# Redis 列表命令

下表列出了列表相关的基本命令：

序号 命令及描述
1 [BLPOP key1 key2 ] timeout (opens new window) 移出并获取列表的第一个元素， 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止。
2 [BRPOP key1 key2 ] timeout (opens new window) 移出并获取列表的最后一个元素， 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止。
3 BRPOPLPUSH source destination timeout (opens new window) 从列表中弹出一个值，将弹出的元素插入到另外一个列表中并返回它； 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止。
4 LINDEX key index (opens new window) 通过索引获取列表中的元素
5 LINSERT key BEFORE|AFTER pivot value (opens new window) 在列表的元素前或者后插入元素
6 LLEN key (opens new window) 获取列表长度
7 LPOP key (opens new window) 移出并获取列表的第一个元素
8 [LPUSH key value1 value2] (opens new window) 将一个或多个值插入到列表头部
9 LPUSHX key value (opens new window) 将一个值插入到已存在的列表头部
10 LRANGE key start stop (opens new window) 获取列表指定范围内的元素
11 LREM key count value (opens new window) 移除列表元素
12 LSET key index value (opens new window) 通过索引设置列表元素的值
13 LTRIM key start stop (opens new window) 对一个列表进行修剪(trim)，就是说，让列表只保留指定区间内的元素，不在指定区间之内的元素都将被删除。
14 RPOP key (opens new window) 移除列表的最后一个元素，返回值为移除的元素。
15 RPOPLPUSH source destination (opens new window) 移除列表的最后一个元素，并将该元素添加到另一个列表并返回
16 [RPUSH key value1 value2] (opens new window) 在列表中添加一个或多个值
17 RPUSHX key value (opens new window) 为已存在的列表添加值

# Redis 集合(Set)

Redis 的 Set 是 String 类型的无序集合。集合成员是唯一的，这就意味着集合中不能出现重复的数据。

集合对象的编码可以是 intset 或者 hashtable。

Redis 中集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)。

集合中最大的成员数为 232 - 1 (4294967295, 每个集合可存储40多亿个成员)。

redis 127.0.0.1:6379> SADD runoobkey redis
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mongodb
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mysql
(integer) 1
redis 127.0.0.1:6379> SADD runoobkey mysql
(integer) 0
redis 127.0.0.1:6379> SMEMBERS runoobkey

1) "mysql"
2) "mongodb"
3) "redis"

在以上实例中我们通过 SADD 命令向名为 runoobkey 的集合插入的三个元素。

# Redis 集合命令

下表列出了 Redis 集合基本命令：

序号 命令及描述
1 [SADD key member1 member2] (opens new window) 向集合添加一个或多个成员
2 SCARD key (opens new window) 获取集合的成员数
3 [SDIFF key1 key2] (opens new window) 返回第一个集合与其他集合之间的差异。
4 [SDIFFSTORE destination key1 key2] (opens new window) 返回给定所有集合的差集并存储在 destination 中
5 [SINTER key1 key2] (opens new window) 返回给定所有集合的交集
6 [SINTERSTORE destination key1 key2] (opens new window) 返回给定所有集合的交集并存储在 destination 中
7 SISMEMBER key member (opens new window) 判断 member 元素是否是集合 key 的成员
8 SMEMBERS key (opens new window) 返回集合中的所有成员
9 SMOVE source destination member (opens new window) 将 member 元素从 source 集合移动到 destination 集合
10 SPOP key (opens new window) 移除并返回集合中的一个随机元素
11 [SRANDMEMBER key count] (opens new window) 返回集合中一个或多个随机数
12 [SREM key member1 member2] (opens new window) 移除集合中一个或多个成员
13 [SUNION key1 key2] (opens new window) 返回所有给定集合的并集
14 [SUNIONSTORE destination key1 key2] (opens new window) 所有给定集合的并集存储在 destination 集合中
15 [SSCAN key cursor MATCH pattern] [COUNT count] (opens new window) 迭代集合中的元素
